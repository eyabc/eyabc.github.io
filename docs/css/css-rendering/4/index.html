caret position & offset#

offset 우리가 css 속성을 주거나 태그의 포함관계를 만들어서 지시를 한다. 지시를 내릴 때, 지시를 추상적으로 내린다.(float, left, ...)

추상적으로 내려도 geometry 계산이 끝나면 fixed Number 로 바뀐다. 이 숫자들을 전부다 offset 이라고 부른다.(고유명사)

원래 CS 에서 offset 은 무엇에서부터 얼마큼 떨어진 이라는 뜻이다. 여기서는 geometry 계산이 끝나서 fixed Number 체계로 바꿔졌을 때의 숫자를 의미하는 고유명사다 변경이 불가능한 읽기 전용이다.

추상적인 명령(우리의 희망사항은)의 수용은 브라우저 마음이다.

브라우저 구형 ie6 에는 float 를 주었을 때 padding 3 이 생기는 계산이 있었다.

실제 그림은 우리가 준 요청대로 계산해서 그릴 것인데(브라우저가) 브라우저 마음대로 그릴것이다. (브라우저 렌더링 시스템) 실제 계산된 수치는 offset 를 조사해야 한다.

브라우저는 효율적인 계산을 하기 위해서 geometyr 계산을 한번에 몰아서 하려고 한다. 한번에 묶어서 계산하는 단위를 프레임이라고 한다. 그림을 그릴 떄 모든 변경요소를 변경만 시켜놓고 재계산을 한번만 시켜주는 타이밍을 프레임이라고 한다. 쌓아있던 큐를 소진하는 것을 flush 라고 한다.

어떤 element 의 offset 를 조사하려고 할려면 즉시 재계산 해야 한다. 큐를 바로 지우고 무조건 재계산을 해야 한다. 한 프레임이 아닌데도 offset 을 요청하면 재계산을 마구마구 때린다 offset 을 요청하면 최적화 로직이 깨져버리고 안쓰는 것이 좋다.

마구 불러도 되는 경우는, geometry 계산이 다 끝났을 때이다. layout 의 offset 를 기준으로 다시 layout 을 그릴려고 하면 한 프레임에 재계산을 여러번 하게 된다.

layout 계산이 없을 때는 많이 불러도 상관 없지만 layout 을 바탕으로 또 layout 을 계산하려고 하면 조심해서 짜야 한다.

offset parent#

offset 을 계산하려면 실제 픽셀값을 계산해야 한다. 컴퓨터의 입장에서는 offset 를 계산하는 가장 기본적인 로직은 상대적인 위치로 모든 수치가 기술되어 있기 때문에 기준점을 찾는 것이 가장 우선된다.

left: 100, top:50 ~ 은 기준점으로 부터라는 의미다. 기준점을 offset parent 라고 한다.

offset parent 는 dom parent 가 아니다.

offset parent 는 어떻게 구하는 것일까?

1. null 인 경우#

  • root, html, body
  • position: fixed (브라우저의 bound box 를 기준으로 그린다)
  • out of dom tree
    • remove child element
    • create element 는 offset parent 가 존재하지 않지만
    • append 후에는 생겨난다.

relative absolute

  • https://www.zerocho.com/category/CSS/post/5864f3b59f1dc000182d3ea1
  • relative 가 static 인 상태를 기준으로 주어진 픽셀만큼 움직였다면,
  • absolute 는 position: static 속성을 가지고 있지 않은 부모를 기준으로 움직입니다. 만약 부모 중에 포지션이 relative, absolute, fixed인 태그가 없다면 가장 위의 태그(body)가 기준이 됩니다.

2. recursive search#

어떤 element 를 찾는 것은 재귀적으로 찾아 들어간다. (ok 는 부모 위치를 위로 계속 찾을 수 있다는 의미)

  • parent.position: fixed = null
    • element 의 부모의 position 이 fixed 일 때 null 이다
  • parent.position:!static = ok
    • static 이 아닌 부모를 찾을 때 까지 위로 올라간다.
    • absolute | relative 이면 중단하고,
    • absolute 의 offset parent 가 될 수 있는 대상은 absolute 이거나 relative 이다
    • absolute 는 내가 위치를 직접 다 지정해야 한다.
      • 어떤 공간(컨테이너)을 기준으로 position 을 만들고 싶으면
      • 기준을 relative 으로 하고,
        • 자식 요소에 absolute 을 해주면 된다.
        • relative 의 역할이 static 의 위치를 조정하는데 쓴다기 보다는 일반적으로 static 안 에 abolute 를 넣기 위한 컨테이너로 사용한다.
    • relative 의 경우는, static 의 위치로 정해진다.
  • body = ok
  • td, ht, table = ok
    • 스펙과 다르게 많은 브라우저에서 테이블의 태그들이 offset parent 로 작동하지 않는다.
    • 안에 div 의 position: relative 속성을 넣어줘야 한다.
  • parent.parent continue,,,

이 공식은 복잡해 보이지만 간단하다.

offset value#

offset parent 를 알았으면 나머지 브라우저가 실제로 계산하는 offset 값을 읽기전용으로 참조할 수 있다.

보이는 곳의 영역#

  • offsetLeft
    • offset parent 의 왼쪽으로 부터 얼마나 떨어져 있느냐
  • offsetTop
  • offsetWidth
    • 실제로 확보된 geometry 크기
  • offsetHeight

진짜 contents 의 영역#

  • offsetScrollTop

  • offsetScrollLeft

    • 스크롤 영역이 생겼을 때, 스크롤의 위치

  • offsetScrollWidth

    • 스크롤 영역이 생겼을 때, 스크롤의 전체크기

  • offsetScrollHeight

absolute#

<div style="width:200px; height: 200px; background: yellow; margin: 100px">
<div style="width: 100px; height: 100px;
position: absolute; background: red">
</div>
<div style="width: 100px; height: 100px;
position: absolute; background: blue; left: 0">
</div>
</div>
  • blue box
    • position: absolute
    • offset parent is <body>

position: absolute 의 left, top 등이 없을 때의 offset 이 offset parent 의 (0, 0) 이 아니다.

  • 부모 element 가 static 이라고 할지라도 DOM 상에 있는 부모의 값을 갖게 된다.
  • red box 는 static 을 준 것 처럼 그려지게 된다.
  • 즉, position: absolute 의 기본값은 static 과 똑같은 위치에 그려지게 된다.

static 에게 left, top 을 준다면?

  • static 은 normal flow 로 그린다.
  • left, top 을 무시한다.

relative 일 때 left, top

  • normal flow 로 그려졌을 때의 차이 값

float 와 position 중 어떤 것이 먼저 적용될까?

  • float 는 새로운 bfc 영역을 만들어 내는 normal flow 에서만 작동한다.
  • 따라서 position: absolute 를 주는 순간 float 는 무의미 해진다.
